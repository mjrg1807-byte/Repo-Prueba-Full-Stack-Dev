Guía completa de HTML

Documento de consulta rápida para el módulo de HTML. Úsalo como referencia mientras programas.

¿Qué es HTML?

HTML (HyperText Markup Language) es el lenguaje que define la estructura y el contenido de una página web. No es un lenguaje de programación.

  • Define estructura
  • Organiza contenido
  • Usa etiquetas semánticas

Estructura básica de un documento HTML

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <title>Título</title>
  </head>
  <body>
  </body>
</html>

Siempre obligatoria. Sin esto el documento no es correcto.

Texto y formato

Títulos

<h1> a <h6> (solo un h1 por página)

Párrafos y énfasis

  • <p> párrafos
  • <strong> importancia
  • <em> énfasis
  • <u> subrayado visual

Enlaces

<a href="https://ejemplo.com" target="_blank">Enlace</a>

Usa target="_blank" solo cuando sea necesario.

Imágenes

<img src="ruta.jpg" alt="Descripción obligatoria">

El atributo alt es obligatorio por accesibilidad.

<figure>
  <img src="foto.jpg" alt="Descripción">
  <figcaption>Texto descriptivo</figcaption>
</figure>

Listas

  • <ul> lista desordenada
  • <ol> lista ordenada
  • <dl> lista de definición

Tablas

<table>
  <tr>
    <th>Título</th>
  </tr>
  <tr>
    <td>Dato</td>
  </tr>
</table>

Contenedores

Div

Contenedor genérico sin significado semántico.

Etiquetas semánticas

  • <header>
  • <nav>
  • <main>
  • <section>
  • <article>
  • <footer>

Formularios

<form>
  <label for="nombre">Nombre</label>
  <input type="text" id="nombre">
</form>

Tipos de input importantes

  • text, number, email, password
  • radio, checkbox
  • date, time, color, range
  • submit, reset

Vídeo y Audio

<video controls>
  <source src="video.mp4" type="video/mp4">
</video>
        
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
</audio>
        
<iframe src="https://www.youtube.com/embed/ID" allowfullscreen></iframe>

Consejos importantes

  • Usa siempre etiquetas semánticas
  • Indenta tu código
  • Usa alt en imágenes
  • Un solo h1 por página
  • No uses HTML para diseño
  • Piensa siempre en accesibilidad
