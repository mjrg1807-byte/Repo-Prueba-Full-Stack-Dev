Asincronía en JavaScript

1. ¿Qué es la asincronía?

JavaScript es un lenguaje single-thread, lo que significa que solo puede ejecutar una tarea a la vez.

La asincronía permite ejecutar tareas que tardan tiempo (como peticiones a internet) sin bloquear el resto del programa.

2. Código síncrono vs asíncrono

Ejemplo síncrono


console.log("Inicio");

for(let i = 0; i < 1000000000; i++) {}

console.log("Fin");

👉 El código se bloquea hasta terminar el bucle.

Ejemplo asíncrono


console.log("Inicio");

setTimeout(() => {
    console.log("Esto aparece después");
}, 2000);

console.log("Fin");

Resultado:


Inicio
Fin
Esto aparece después

👉 JavaScript sigue ejecutando el código mientras espera.

3. ¿Qué es una Promesa?

Una promesa es un objeto que representa un valor que estará disponible en el futuro.

Estados de una promesa

  • ⏳ Pending → pendiente
  • ✅ Fulfilled → resuelta
  • ❌ Rejected → rechazada

4. Crear una promesa


const promesa = new Promise((resolve, reject) => {

    let exito = true;

    if(exito){
        resolve("Todo salió bien");
    } else {
        reject("Hubo un error");
    }

});

Explicación

  • resolve() → la promesa salió bien
  • reject() → ocurrió un error

5. Consumir una promesa


promesa
    .then((resultado) => {
        console.log(resultado);
    })
    .catch((error) => {
        console.log(error);
    });

Explicación

  • .then() → se ejecuta si todo sale bien
  • .catch() → se ejecuta si hay error

6. Async y Await

Async/Await es una forma moderna y más legible de trabajar con promesas.

Ejemplo


async function ejecutarPromesa(){

    try {

        const resultado = await promesa;

        console.log(resultado);

    } catch(error){

        console.log(error);

    }

}

ejecutarPromesa();

Conceptos importantes

  • async → convierte una función en asíncrona
  • await → espera el resultado de una promesa
  • try/catch → manejo de errores

💡 Aunque parece síncrono, sigue siendo asincrónico.

7. Fetch

Fetch permite hacer peticiones HTTP a servidores o APIs.

Ejemplo básico con .then()


fetch("https://jsonplaceholder.typicode.com/users")

    .then((response) => {
        return response.json();
    })

    .then((data) => {
        console.log(data);
    })

    .catch((error) => {
        console.log(error);
    });

¿Qué ocurre aquí?

  • fetch() → hace la petición
  • response.json() → convierte la respuesta a JSON
  • data → contiene los datos finales

8. Fetch con Async/Await

Forma moderna y profesional


async function obtenerUsuarios(){

    try {

        const response = await fetch("https://jsonplaceholder.typicode.com/users");

        const data = await response.json();

        console.log(data);

    } catch(error){

        console.log("Error:", error);

    }

}

obtenerUsuarios();

9. Flujo real de Fetch


1. fetch() hace la petición
2. El servidor responde
3. response.json() convierte los datos
4. Ya puedes usar la información

10. Error típico de principiantes

❌ Incorrecto


const datos = fetch("https://api.com");

console.log(datos);

👉 Esto devuelve una Promise, NO los datos finales.

✅ Correcto


const response = await fetch("https://api.com");

const data = await response.json();

console.log(data);

11. Diferencia entre return y console.log

❌ Incorrecto


function sumar(a, b){
    console.log(a + b);
}

👉 Solo imprime el resultado.

✅ Correcto


function sumar(a, b){
    return a + b;
}

const resultado = sumar(2, 3);

console.log(resultado);

👉 La función devuelve el valor y luego decides qué hacer con él.

12. Resumen general

  • JavaScript usa asincronía para no bloquear el programa
  • Las promesas representan valores futuros
  • .then() y .catch() manejan promesas
  • async/await hace el código más limpio
  • fetch permite obtener datos de APIs

13. Cómo pensar la asincronía

❌ NO pienses: 


"Haz esto y espera parado"

✅ Piensa: 


"Haz esto, sigue trabajando y cuando termine vuelve"

14. Ejercicio recomendado

  • Haz un fetch a una API pública
  • Muestra los datos en consola
  • Hazlo primero con .then()
  • Luego conviértelo a async/await
